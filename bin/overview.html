Flight Booking System

Overview

The Flight Booking System is a comprehensive Java application for managing flight bookings, customers, and administrative operations. The system provides both command-line and graphical user interfaces with role-based authentication and authorization.

Key Features

  • Authentication & Authorization: Secure login system with Admin and Customer roles
  • Flight Management: Create, update, delete, and search for flights
  • Customer Management: Register and manage customer accounts
  • Booking Operations: Book, cancel, and update flight reservations
  • Data Persistence: File-based storage for all system data
  • Dual Interface: Both CLI and GUI modes available

Architecture

The application follows a layered architecture with clear separation of concerns:

  • Model Layer: Core business entities (Flight, Customer, Booking, User)
  • Command Layer: Command pattern implementation for all operations
  • Data Layer: Data persistence and loading mechanisms
  • Presentation Layer: GUI and CLI interfaces
  • Authentication Layer: Security and user management

Getting Started

To use the system, run the Main class for CLI mode or use the loadgui command to launch the graphical interface. Default admin credentials are provided in the documentation.

Design Patterns

  • Command Pattern: All operations are encapsulated as command objects
  • Singleton: AuthenticationService ensures single authentication instance
  • Factory: CommandParser creates appropriate command instances
  • Data Access Object: DataManager classes handle persistence
